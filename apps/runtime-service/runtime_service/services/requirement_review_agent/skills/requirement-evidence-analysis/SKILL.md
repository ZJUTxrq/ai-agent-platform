# 需求证据分析

使用本 skill 整理当前可用需求证据，为后续评分提供事实基础。

## 输入

- 用户输入的需求文本
- 当前轮上传 PDF/图片的多模态摘要或解析结果
- 用户明确给出的目标模块、业务规则、约束条件、验收标准

## 分析重点

- 业务目标和用户价值
- 功能范围和边界
- 主流程、分支流程、异常流程
- 业务规则、字段规则、状态流转、权限规则
- 可观察输入、输出、状态和验收标准
- 接口、数据、外部依赖、边界条件和假设

## 输出要求

输出中间分析，不要给最终门禁结论。

必须整理：

- `requirement_summary` 初稿
- 已明确的需求事实
- 证据不足或存在歧义的点
- 评分时需要重点关注的风险

本 skill 不生成正式测试用例。
