# 业务流程需求评审检查清单(business-flow-review)

本 skill 用于涉及流程、状态流转、审批的需求。目标:判断状态机与流程规则是否完整、无歧义。

## 必查项

1. **状态机完整性**:是否枚举了全部状态与允许的迁移?
   对任意"状态 X 收到操作 Y"能否唯一确定结果?不能即为发现项。
2. **非法迁移**:未定义的状态迁移(如已完成订单再次取消)期望行为是否说明?
3. **异常路径**:回退、撤销、驳回、超时的触发条件与目标状态是否定义?
   驳回后重新提交是回到原节点还是重走全流程?
4. **并发与重复**:同一单据被两人同时操作、重复提交的行为是否定义?
5. **边界场景**:流程为空/单节点、跳过节点、代理审批等边界是否覆盖?
6. **角色与时序**:每个节点的操作角色、时限、超时后的自动行为是否明确?

## 知识库对照

- 以流程名/单据名/状态名调用 `query_project_knowledge`(`only_need_context: true`),
  核对知识库中已有的状态流转与流程规则,检查本需求是否与现行流程矛盾;
- 矛盾处按评分 skill 的矛盾/硬冲突规则处理,并逐字引用来源。

## 发现项输出要求

每条发现项 = 问题描述 + 证据(需求原文或知识库引用)+ 对流程正确性/测试设计的影响。
发现项挂靠 manifest 声明的评分维度,由 `requirement-quality-scoring` 阶段统一扣分;
本 skill 不直接给分。
