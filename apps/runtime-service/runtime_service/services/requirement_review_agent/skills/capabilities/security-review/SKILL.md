# 安全需求评审检查清单(security-review)

本 skill 用于涉及鉴权、权限、敏感数据的需求。目标:判断需求是否把安全相关行为定义到可测试的程度。

## 必查项

1. **鉴权与会话**:登录态如何建立与失效?token/session 的有效期、续期、并发登录行为是否定义?
2. **权限模型**:涉及哪些角色?每个操作的权限边界是否明确?
   必须能回答"角色 A 访问角色 B 的数据会发生什么"——回答不了即为发现项(越权场景未定义)。
3. **敏感数据**:涉及的个人信息/敏感字段是否指明存储加密、传输保护、展示脱敏、日志脱敏要求?
4. **失败与锁定**:认证失败的提示、重试限制、锁定策略是否定义?
5. **审计**:关键操作是否要求留痕?留痕内容与可追溯性是否说明?
6. **合规**:采集个人信息是否说明用途告知与授权环节?

## 知识库对照

- 以"鉴权/权限/密码策略/脱敏"等为关键词调用 `query_project_knowledge`
  (`only_need_context: true`),核对项目已有安全基线与本需求是否冲突;
- 与安全基线冲突属于高危矛盾,按评分 skill 的硬冲突规则处理,并逐字引用来源。

## 发现项输出要求

每条发现项 = 问题描述 + 证据(需求原文或知识库引用)+ 安全影响。
发现项挂靠 manifest 声明的评分维度,由 `requirement-quality-scoring` 阶段统一扣分;
本 skill 不直接给分。
