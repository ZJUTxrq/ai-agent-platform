# 接口需求评审检查清单(api-review)

本 skill 用于涉及接口/API 的需求。目标:判断需求提供的接口信息是否足以支撑接口测试设计。

## 必查项

1. **字段契约**:每个请求/响应字段是否给出类型、必填性、长度/取值范围、枚举值?
   缺少任何一项都构成发现项。
2. **错误码与异常响应**:是否定义了失败场景的错误码、错误信息结构?
   只有正常路径没有异常路径的接口需求,必须列为发现项。
3. **幂等性**:写操作(创建/支付/提交类)是否说明重复请求的行为?
4. **鉴权与调用方**:是否说明接口的鉴权方式与允许的调用方?
   (涉及权限细节的问题移交 security-review,本项只查"是否提及")
5. **版本与兼容**:改动已有接口时,是否说明对存量调用方的兼容策略?
6. **分页/排序/过滤**:列表类接口是否定义分页参数与默认值?
7. **超时与重试**:对外依赖接口是否说明超时与重试语义?

## 知识库对照

- 用接口名/资源名调用 `query_project_knowledge`(`only_need_context: true`),
  检查知识库中已有的接口约束、字段规则与本需求是否一致;
- 不一致处按评分 skill 的矛盾/硬冲突规则处理,并逐字引用来源。

## 发现项输出要求

每条发现项 = 问题描述 + 证据(需求原文或知识库引用)+ 对测试设计的影响。
发现项挂靠 manifest 声明的评分维度,由 `requirement-quality-scoring` 阶段统一扣分;
本 skill 不直接给分。
