# 知识库引用核验(quote-verify)

本 skill 在 `requirement-quality-scoring` 阶段执行,为硬冲突判定提供代码级核验。

## 规则

1. 在把任何知识库引用作为**硬冲突依据**之前,必须对该引用调用 `verify_quote` 工具
   (传入 `project_id`、引用原文 `quote`、声明的来源文档 `source_document`)。
2. `verified: true` 的引用才能作为硬冲突依据。
3. `verified: false` 的引用**不得**用于判定硬冲突:
   - 若确认矛盾存在但引用不准确,先修正引用原文后重新核验;
   - 无法修正时,降级为待澄清项并按普通矛盾扣分,不触发硬冲突红线。
4. 工具返回 `knowledge_mcp_unavailable` 等错误时,视为无法核验,
   同样不得判定硬冲突;在评审报告中说明知识库核验不可用。
5. 普通(非硬冲突)引用不强制核验,但被质疑时应能通过 `verify_quote` 复核。

## 目的

硬冲突会直接把门禁压为 blocked,是影响最大的评审结论;
本核验确保每一条否决理由都逐字存在于知识库原文中,杜绝虚构引用。
